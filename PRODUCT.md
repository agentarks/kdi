# PRODUCT.md — kdi

## Product

**kdi** is a multi-agent Kanban dispatch system for coding agents. It tracks tasks, boards, assignments, and lifecycle states for autonomous and semi-autonomous agents working on software projects.

## Users

Primary users are developers and platform operators who:
- Run coding agents locally or in CI
- Need to track agent work across multiple boards and workflows
- Want a lightweight operator UI alongside the CLI

## Use context

- Used in terminal-heavy environments, IDEs, and browsers
- Often viewed during standups, triage, and dispatch sessions
- Needs to feel like a tool, not a dashboard

## Tone

- Direct, functional, unpretentious
- Confident without being loud
- The UI should feel intentional and opinionated

## Anti-references

- Generic SaaS dark-blue dashboard
- Gradient hero sections
- Soft, rounded, pastel “friendly” enterprise UI
- Excessive whitespace that hides information

## Strategic principles

- The tool is infrastructure, not a product showcase
- Information density is a feature
- Every element must earn its place
- Design serves the operator, not the portfolio

## Register

Product (tool UI). The design serves the product; the product is not the design.
